# 상품 정보 동기화 가이드

홈페이지 상품 목록(`lib/products.ts`)을 **재고 상품총괄표 엑셀** 하나로 관리합니다.
가격·신상품을 엑셀에만 입력하면, 동기화 스크립트가 홈페이지 상품 목록을 자동 생성합니다.

## 데이터가 사는 곳 (역할 분담)

| 데이터 | 위치 | 누가 관리 |
|--------|------|-----------|
| 상품명·코드·**가격**·사이즈·카테고리, 어떤 상품이 존재하는지 | 재고 상품총괄표 엑셀 (`C:\AI\inventory_dashboard\data\master\상품총괄표_*.xlsx`) | 사장님 (매월 갱신, **단일 원천**) |
| 이미지·구수(holes)·발송메모·스토어 표시명/사이즈 | 홈페이지 `data/product-presentation.json` | 홈페이지 표시용 (가끔 손봄) |
| 최종 상품 목록 | 홈페이지 `lib/products.ts` | **자동 생성 (직접 수정 금지)** |

> `lib/products.ts` 는 이제 자동 생성 파일입니다. 직접 고치지 말고, 엑셀이나 `product-presentation.json` 을 고친 뒤 동기화하세요.

## 사용법

### 1. 가격 변경 / 신상품 추가
1. 재고 상품총괄표 엑셀에서 가격을 바꾸거나 새 상품 행을 추가합니다. (가격은 반드시 입력 — 가격 없으면 미등록 처리)
2. 홈페이지 폴더에서 **`상품동기화.bat`** 더블클릭.
3. 화면에 `➕ 신규 / 💲 가격변경 / ⚠️ 확인 필요` 요약이 뜹니다.
4. 결과가 맞으면 git 커밋·푸시 → Vercel 자동 배포.

### 2. 신상품 이미지/표시 정보 넣기
새 상품은 동기화하면 **회색 플레이스홀더**로 먼저 올라가고 `⚠️ 이미지 없음` 경고가 뜹니다.
실제 사진을 넣으려면:
1. 썸네일 이미지를 `public/thumbnails/` 에 저장 (예: `브레드나이프.png`)
2. `data/product-presentation.json` 에 해당 상품코드 항목 추가:
   ```json
   "07020": { "id": "WP49200", "note": "안전봉투 발송", "image": "/thumbnails/브레드나이프.png" }
   ```
3. 다시 `상품동기화.bat` 실행.

`product-presentation.json` 에 넣을 수 있는 표시 전용 필드:
`id`(상품ID, 없으면 바코드/코드로 자동), `name`(스토어 표시명, 없으면 엑셀명), `size`(엑셀에 없을 때),
`holes`(구수), `holeSize`(구 크기), `note`(발송메모), `image`(썸네일경로), `imageColor`(이미지 없을 때 카드 색).

## 제외 규칙 (스토어에 안 올라가는 상품)
- 카테고리가 **기타** 인 상품 (차량용컵홀더, 반품회송 등) — `scripts/sync-config.json` 의 `excludeCategories`
- **가격 0원** 상품 (관리용·비매품)
- 특정 코드를 빼려면 `scripts/sync-config.json` 의 `excludeCodes` 에 추가

## 동기화하지 않고 변경점만 미리보기
```
node scripts/sync-products.mjs --check
```

## 마스터 엑셀 경로 바꾸기
`scripts/sync-config.json` 의 `masterDir` 수정. 파일명에 날짜가 있어도 (`상품총괄표_*.xlsx`) 가장 최근 파일을 자동으로 찾습니다.
